# Use-Case Prioritization Checklist

## 1. Define the business problem
- What is the actual problem?
- Who experiences it?
- How often does it occur?
- What is the cost of leaving it unsolved?

## 2. Check AI fit
- Does the task involve pattern recognition, summarization, classification, generation, or retrieval?
- Is the problem narrow enough to test?
- Is there enough input data or process clarity?
- Can outputs be reviewed reliably by humans?

## 3. Estimate value
- Will this save time, reduce cost, improve quality, or increase speed?
- Which team benefits first?
- Is the expected value visible within a reasonable timeframe?
- Does the use case support a larger strategic goal?

## 4. Assess feasibility
- Are the workflow and ownership clear?
- Are the required tools and data accessible?
- Are there governance, legal, or compliance concerns?
- Can the use case be piloted safely?

## 5. Decide priority
- Is this a high-value, low-friction opportunity?
- Is it suitable for a pilot, a phased rollout, or not yet?
- What would success look like in 30 to 60 days?
